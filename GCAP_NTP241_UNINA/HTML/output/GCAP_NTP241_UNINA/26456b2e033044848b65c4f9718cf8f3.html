(C-05) Geolocation System

ControlEntity

GCAP_NTP241_UNINA > GCAP_NTP241_UNINA > STPA Analysis > Control Structure > (C-05) Geolocation System

No description.

Content

  • The Geolocation System shall reject GNSS data with low signal confidence or integrity values.
  • The Geolocation System shall switch to fallback localization (e.g., IMU, compass) upon GNSS loss or detected spoofing.
  • The Geolocation System shall continuously validate GNSS inputs using sensor fusion with onboard sensors (IMU, compass, pressure sensor).
  • The Geolocation System shall not send positional data to the Autopilot unless sensor fusion confidence thresholds are met.
  • The Geolocation System shall log and report any positional uncertainty or inconsistency exceeding predefined thresholds.
  • The Geolocation System shall monitor for inconsistencies between predicted and actual position changes.
  • The Geolocation System shall exchange data with other components across a secure channel
  • The Geolocation System shall ensure integrity of the firmware it runs
  • The Geolocation System shall enforce least-privilege execution
  • (R-09) Calculate current drone position based on GNSS and sensor fusion
  • (R-10) Validate the integrity and accuracy of received GNSS data
  • (R-11) Provide position data to the Autopilot in real time
  • (R-12) Detect anomalies in GNSS signal (e.g., spoofing or jamming) and raise alarms
  • (R-13) Use fallback localization (e.g., from IMU or Compass) when GNSS is unavailable or untrusted
  • (R-14) Implement firmware integrity validation (e.g., signed updates, hash checks)
  • (R-15) Exchange geolocation info with other components over a secured channel
  • Feedback: (C-06) GNSS Source -> (C-05) Geolocation System
  • Control: (C-05) Geolocation System -> (C-03) Autopilot
