(LS-02) Tampering of Control Action CA3 via FDI Attack

LossScenario

GCAP_NTP241_UNINA > GCAP_NTP241_UNINA > STPA Analysis > Loss Scenarios > (LS-02) Tampering of Control Action CA3 via FDI Attack

No description.

Causal Factors

  • [UCA-18] Control Input: Inadequate

Control Action

  • (CA-03) Drone Position [BUS]

Violated Constraints

  • (SC-05) Navigation calculations shall be validated against sensor fusion confidence thresholds.
  • (SC-08) Information among drone components shall happen across secure channels

Countermeasures

  • The Geolocation System shall reject GNSS data with low signal confidence or integrity values.
  • The Geolocation System shall continuously validate GNSS inputs using sensor fusion with onboard sensors (IMU, compass, pressure sensor).
  • The Geolocation System shall not send positional data to the Autopilot unless sensor fusion confidence thresholds are met.
  • The Geolocation System shall monitor for inconsistencies between predicted and actual position changes.
  • The Geolocation System shall exchange data with other components across a secure channel
  • The Geolocation System shall ensure integrity of the firmware it runs
  • The Geolocation System shall enforce least-privilege execution

Hazards

  • (H-03) Loss of GPS signal
  • (H-05) Loss of planned route due to imprecise GPS data

Unsafe Control Actions

  • (UCA-18) The geolocation system provides wrong or inconsistent coordinates.
