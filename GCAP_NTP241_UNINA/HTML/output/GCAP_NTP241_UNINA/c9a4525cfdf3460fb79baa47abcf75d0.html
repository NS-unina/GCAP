(SC-03) The drone shall switch to inertial navigation and trigger RTH (Return-To-Home) upon GPS signal loss.

SystemConstraint

GCAP_NTP241_UNINA > GCAP_NTP241_UNINA > STPA Analysis > Hazards & Constraints > (SC-03) The drone shall switch to inertial navigation and trigger RTH (Return-To-Home) upon GPS signal loss.

No description.

Hazards

  • (H-03) Loss of GPS signal

Responsibilities

  • (R-06) Integrate position data into control logic.
  • (R-10) Validate the integrity and accuracy of received GNSS data
  • (R-12) Detect anomalies in GNSS signal (e.g., spoofing or jamming) and raise alarms
  • (R-13) Use fallback localization (e.g., from IMU or Compass) when GNSS is unavailable or untrusted

Loss Scenarios

  • (LS-01) Wrong GPS Data

Unsafe Control Actions

  • (UCA-13) GNSS source fails to send coordinates to the GPS module (signal loss or transmission error).
  • (UCA-14) GNSS provides incorrect satellite data (e.g., due to spoofing, drift, or error in ephemeris data).
  • (UCA-17) The geolocation system fails to send any position data to the autopilot.
  • (UCA-20) Position is calculated using a degraded or unverified GNSS signal, leading to a faulty position fix.
