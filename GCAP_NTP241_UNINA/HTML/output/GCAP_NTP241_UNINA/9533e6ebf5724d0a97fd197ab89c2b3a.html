(SC-05) Navigation calculations shall be validated against sensor fusion confidence thresholds.

SystemConstraint

GCAP_NTP241_UNINA > GCAP_NTP241_UNINA > STPA Analysis > Hazards & Constraints > (SC-05) Navigation calculations shall be validated against sensor fusion confidence thresholds.

No description.

Hazards

  • (H-05) Loss of planned route due to imprecise GPS data

Responsibilities

  • (R-05) Compute flight dynamics from sensor and mission data.
  • (R-09) Calculate current drone position based on GNSS and sensor fusion
  • (R-11) Provide position data to the Autopilot in real time
  • (R-13) Use fallback localization (e.g., from IMU or Compass) when GNSS is unavailable or untrusted

Loss Scenarios

  • (LS-01) Wrong GPS Data
  • (LS-02) Tampering of Control Action CA3 via FDI Attack

Unsafe Control Actions

  • (UCA-19) The position is sent with excessive latency, affecting flight path corrections.
  • (UCA-15) GPS module misinterprets or misuses otherwise correct satellite data.
  • (UCA-16) GNSS signal is significantly delayed, affecting real-time position estimation.
  • (UCA-14) GNSS provides incorrect satellite data (e.g., due to spoofing, drift, or error in ephemeris data).
  • (UCA-18) The geolocation system provides wrong or inconsistent coordinates.
  • (UCA-20) Position is calculated using a degraded or unverified GNSS signal, leading to a faulty position fix.
