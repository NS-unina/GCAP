(H-01) Uncontrolled Emergency Landing

Hazard

GCAP_NTP241_UNINA > GCAP_NTP241_UNINA > STPA Analysis > Hazards & Constraints > (H-01) Uncontrolled Emergency Landing

No description.

System-Level Constraints

  • (SC-01) The drone shall initiate controlled emergency procedures upon detection of flight instability.
  • (SC-08) Information among drone components shall happen across secure channels

Losses

  • (L-01) Loss of Lives
  • (L-02) Loss of Drone

Loss Scenarios

  • (LS-03) Tampering of Control Action CA2 via Command INjection Attack

Unsafe Control Actions

  • (UCA-04) Commands are sent despite no operator input (e.g., noise interpreted as signal).
  • (UCA-07) Telemetry data is delayed, leading to outdated operator decisions.
  • (UCA-08) Incoherent sensor status is reported (e.g., IMU and GPS data mismatch).
  • (UCA-09) IMU fails to send motion/acceleration data to the Autopilot.
  • (UCA-10) IMU sends corrupted, biased, or uncalibrated data (e.g., due to sensor drift or malfunction).
  • (UCA-11) Autopilot misinterprets valid IMU data, applying wrong compensation or control decisions.
  • (UCA-12) IMU data is received too late to ensure timely correction of flight dynamics.
  • (UCA-01) The RC fails to send commands when user input is given (e.g., stick movement not transmitted).
